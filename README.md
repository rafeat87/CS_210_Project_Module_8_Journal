# CS_210_Project_Module_8_Journal
Module 8 Journal submission
Corner Grocer Item Tracking Program
Project Summary

This project solves the problem of tracking grocery item purchases and calculating item frequency. The program reads an input file of purchased items, counts how often each item appears, and allows the user to search, display, and visualize purchase frequencies using a menu-driven interface. It also creates a backup data file for record keeping.

What I Did Well

One of my strongest areas in this project was implementing object-oriented programming using a class with public and private sections. I also successfully used file handling and the STL map container to efficiently store and retrieve item frequency data. The program includes input validation to prevent crashes and infinite loops.

Areas for Improvement

While the program functions correctly, it could be enhanced by:

Adding case-insensitive searching

Improving user interface formatting

Adding error handling for missing input files

Separating implementation into .h and .cpp files for better modularity

These improvements would make the program more efficient, secure, and professional.

Most Challenging Parts

The most challenging part was implementing the menu system with proper input validation. Initially, invalid input caused infinite loops. I overcame this by using cin.fail(), cin.clear(), and cin.ignore() to properly handle incorrect user input. I used course materials, documentation, and debugging techniques to solve this issue.
